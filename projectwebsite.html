About the Project

The CampusCourier project is designed to bring smart delivery solutions to university campuses. We aim to create a delivery robot that operates through an LTE-controlled system in its initial phase, with the goal of evolving it into a fully autonomous system capable of handling deliveries across campus.

Project Phases:

  • Phase 1: Remote-controlled system using LTE networks for real-time control.
  • Phase 2: Transition to an AI-powered autonomous delivery system with obstacle detection and navigation capabilities.

Research & Development

The project explores the integration of LTE communication with robotic systems and examines the performance and challenges of autonomous navigation. Future research papers and thesis work will address the key findings from both phases of development.

Phase 1: Remote-Control

We are focusing on building a robust LTE-controlled system, tackling the challenges of latency and real-time communication.

Phase 2: Autonomous Navigation

Our autonomous system will use AI and sensor fusion technologies to safely navigate the campus. We are exploring AI algorithms to optimize the robot’s efficiency in real-world scenarios.

Technology Stack

Hardware:

  • Microcontrollers (Arduino, Raspberry Pi)
  • LTE modules for real-time control
  • LiDAR and GPS for autonomous navigation

Software:

  • Python for programming control systems
  • ROS (Robot Operating System) for autonomy
  • AI and machine learning for decision-making

Progress and Milestones

  • Month 1-3: Develop remote control via LTE networks.
  • Month 4-6: Complete tests for range and latency.
  • Month 7-9: Begin autonomous system development.
  • Month 10-12: Full autonomous testing and delivery integration.
